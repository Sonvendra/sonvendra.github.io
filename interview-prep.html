Q2. What is RxJS? +

RxJS is a library for composing asynchronous and event-based programs using Observable sequences.

  • Core type: Observable
  • Satellite types: Observer, Subject, Scheduler
  • Operators: map, filter, reduce, every, etc.
  • Handles async events as collections
Q3. What is the purpose of RxJS? +
  • Manage asynchronous data streams
  • Works with:
    1. HTTP calls
    2. Callbacks
    3. Events
  • Provides an extensive list of operators to transform data
Q4. What are Observables in RxJS? +

Observables act as an interface to handle:

  • Asynchronous operations
  • Streams of data
  • Events and HTTP responses
What is forkJoin? +

forkJoin is an RxJS operator that:

  • Runs multiple observables in parallel
  • Waits for all to complete
  • Emits only once with last values

const finalVal = forkJoin([json1, json2, json3]);
  • ‚ùå Never emits if any observable never completes
  • ‚ùå Errors immediately if any observable errors
Q5. What is RegExp? +

Regular Expressions are patterns used to match text in strings.

const a = 'anil goes to school';
const regex = /[aeiou]/gi;
const vowelsCount = a.match(regex);
Q6. What is NgRx? +

NgRx is a reactive state management library for Angular inspired by Redux.

  • Single-direction data flow
  • Centralized Store
  • Isolation of side effects
Core building blocks:
  1. Actions
  2. Reducers
  3. Selectors
  4. Store

// app.module.ts
StoreModule.forRoot({ hero: heroReducer });


// Dispatch
this.store.dispatch(setData({ data: 'Test' }));


// Select
this.hero$ = this.store.select(selectHeroData);
